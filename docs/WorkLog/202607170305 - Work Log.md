### PROCESS SHIFT — Build the Marker System

The system will generate a series of uniquely coded QR markers. The markers will be printed separately, cut out, and pasted onto the flyer template.

Each QR code opens:

`/m/{marker-code}`

Initial test markers:

- `test-001`
- `test-002`
- `test-003`

A marker is an entrance. It will later be connected to a Point.

## Core process

1. **Point Creator provides Point information**
    
    Name, description, type, creator, controller, and status.
    
2. **Cloudflare creates the Point**
    
    The Point is stored in D1 and receives a unique Point ID.
    
3. **The system creates an Entrance**
    
    The Entrance receives a unique marker code:
    
    ```
    /m/test-001
    ```
    
4. **The system generates a QR code**
    
    The QR contains the Entrance URL. It is printed, cut out, and pasted onto the flyer.
    
5. **The Entrance is connected to the Point**
    
    ```
    QR code → marker address → Point
    ```
    
6. **A visitor scans the marker**
    
    The system resolves the marker code and displays the Point interface.
    
7. **The visitor interacts with the Point**
    
    The interface can display information and accept new events.
    
8. **Events become part of the Point’s history**
    
    Text, photographs, audio, movement, observations, or other interactions can be recorded as Point-specific events.
