# Three Channels of Development

## 1. Point System — Information

1. Build the event logger.
2. Create a Point.
3. Generate a changing address for the Point.
4. Connect the address to a physical QR code.
5. Define Point membership and access.
6. Attach every submission to:

   * Point of origin
   * Author
   * Time
7. Process submissions into local digests.
8. Define controlled information movement between Points.
9. Rotate addresses and retire Points.
10. Convert retired Points into archives.
11. Create a locally owned publisher for each address.
12. Create the neighborhood-wide publisher.
13. Register durable references, authorship, and ownership on-chain.
14. Develop archives and digests into marketable data products.

## 2. Economic Union — Value

1. Define union membership.
2. Define the unit of value: $1 deposited = $1 claim.
3. Define the 14-year property-liquidation agreement.
4. Establish the entity selling property equity to the union.
5. Select the first property.
6. Define how member deposits purchase property equity.
7. Separate union value from property use and occupancy.
8. Define who absorbs property risk.
9. Define the minimum property collateral required.
10. Define withdrawal and transfer restrictions.
11. Define loss seniority: first-in, first-out.
12. Create the member ledger.
13. Create the blockchain contracts for deposits, transfers, and final claims.
14. Build simple ways for members to exchange union value.
15. Develop local businesses that accept union value.
16. Create systems for members to copy and operate those businesses.
17. Liquidate the property after 14 years.
18. Distribute the proceeds to the final value holders.

## 3. Church of the Dead Space Monkey — Transmission

1. Define what information the Church receives.
2. Collect Point archives, publications, and union history.
3. Preserve authorship, origin, and ownership records.
4. Process the collected material into a transmissible archive.
5. Define what remains private during the union’s life.
6. Define the trigger for final transmission.
7. Verify that the union has been liquidated.
8. Launch the completed data archive.
9. Close the local transmission process.
10. Move on to the next community or cycle.

The immediate build order is:

1. **Point:** finish the Point, address, QR, and event-record structure.
2. **Union:** write the value, collateral, risk, and liquidation rules on paper.
3. **Church:** define what it collects now; postpone the final transmission machinery.
